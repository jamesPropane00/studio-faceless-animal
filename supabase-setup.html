Supabase Setup
Step by Step

Follow these exact steps to wire up live chat, private messages, and the member database to your Faceless Animal Studios platform.

Part 1 — Create Your Supabase Project

1

Go to supabase.com and sign in

Create a free account at supabase.com. Click "New Project" and fill in your project name (e.g. faceless-animal-studios), set a strong database password, and choose the US East region.

Open Supabase Dashboard →
2

Copy your Project URL and Anon Key

In your project dashboard go to Settings → API. You need two values:

ValueWhere to find itEnvironment variable
Project URLSettings → API → Project URLSUPABASE_URL
Anon / Public keySettings → API → anon publicSUPABASE_ANON_KEY
Already done. Your project is already connected — these values are in your Replit environment secrets (SUPABASE_URL and SUPABASE_ANON_KEY). This step is only needed if you're setting up a fresh project.
3

Add secrets to Replit

In Replit, open the Secrets panel (lock icon in the left sidebar). Add two secrets:
SUPABASE_URL = your project URL
SUPABASE_ANON_KEY = your anon key

Then restart the dev server. The server writes assets/js/env.js from these secrets so the frontend connects automatically.

Part 2 — Run SQL Migrations (Database Tables)

4

Open SQL Editor in Supabase

In your Supabase Dashboard, click SQL Editor in the left sidebar. You'll paste and run SQL here.

5

Create the Radio Chat table (messages)

This table powers the live radio chat room. Paste this into SQL Editor and click Run:

-- Radio chat messages table create table if not exists messages ( id uuid primary key default gen_random_uuid(), room_name text not null, username text not null, message text not null, created_at timestamptz not null default now() ); alter table messages enable row level security; create policy "Public read" on messages for select using (true); create policy "Public insert" on messages for insert with check ( length(message) > 0 and length(message) <= 300 ); create index if not exists messages_room_created on messages (room_name, created_at desc);
6

Enable Realtime for the messages table

Run this SQL (or enable it via the Dashboard UI):

alter publication supabase_realtime add table messages;

OR via Dashboard: Go to Database → Replication → find the messages table → toggle it ON. This activates the live chat so new messages appear instantly.

7

Create Member Accounts + DM Messages tables

This is the full member platform schema. Open the file supabase/migrations/002_member_platform.sql in your Replit project, copy the entire contents, paste into SQL Editor, and click Run.

This creates:

  • member_accounts — all platform members, their plan, status, page slug
  • dm_messages — private 1:1 messages with Realtime enabled
File location: In Replit, look for supabase/migrations/002_member_platform.sql in the file tree. The file is complete and ready to paste.

Part 3 — Enable Realtime (Live Updates)

8

Enable Realtime for all three tables

Run this block in SQL Editor to enable live updates for chat and DMs:

alter publication supabase_realtime add table messages; alter publication supabase_realtime add table dm_messages;

OR via Dashboard UI: Go to Database → Replication and toggle ON for messages and dm_messages.

Part 4 — Verify Everything Works

9

Check the radio chat is live

Open radio.html on your site and sign in. Try sending a chat message. If it appears in the feed — Supabase is connected and working. Open a second browser window and confirm the message appears there too (Realtime working).

10

Check the member dashboard loads your data

Sign in at login.html with any username. Then go to dashboard.html. Your member account should appear in the member_accounts table in Supabase (visible in Table Editor → member_accounts). That confirms the sync is working.

11

Test private messages

Go to messages.html, click "+ New Message", send a DM to any other username. Check Supabase Table Editor → dm_messages and confirm the row was inserted. Open messages.html in a second browser signed in as the other user — the message should appear in real time.

12

Check the floating radio widget chat

Open any page other than radio.html (like the homepage). The radio bar at the bottom should appear. Click the ↑ button to expand the panel. Click the 💬 Chat tab. Chat messages from radio.html should appear here too — they all come from the same messages table, room radio.

Admin — Updating Member Plans Manually

★

When someone pays — upgrade their plan in Supabase

Go to Supabase → Table Editor → member_accounts. Find the row by username. Update plan_type and member_status:

-- Upgrade a member to Starter plan update member_accounts set plan_type = 'starter', member_status = 'active' where username = 'their_username'; -- Upgrade to Pro update member_accounts set plan_type = 'pro', member_status = 'active' where username = 'their_username'; -- Platform Access only ($5/mo) update member_accounts set plan_type = 'access', member_status = 'active' where username = 'their_username';

Valid plan_type values: free | access | starter | pro | premium
Valid member_status values: free | active | paused | suspended

★

When a member's page is built — update page_slug and page_status

Once you build a member's artist/creator page at e.g. artists/their-slug.html, update their record:

update member_accounts set page_slug = 'their-slug', page_status = 'live' where username = 'their_username';

Their dashboard will then show a "View My Page →" link pointing to their live page.

You're All Set.

Once those SQL migrations are run and Realtime is enabled, the platform is fully live. Radio chat, private messages, and member dashboards all run off Supabase automatically.
